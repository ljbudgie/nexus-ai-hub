# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public issue.
2. Email the maintainers or use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) feature.
3. Include a description of the vulnerability and steps to reproduce it.

We will acknowledge receipt within 48 hours and aim to release a fix promptly.

## Security Best Practices

- Never commit secrets, API keys, or credentials to the repository.
- Keep dependencies up to date.
- Use environment variables for sensitive configuration.
